### **Genesis Training Solutions**

**Project Sprint Instruction Sheet**

**Project:** Smart Plot Investment Portal
**Sprint Duration:** March 9 – March 13
**Sprint Goal:** Establish the core database structure and implement the initial backend authentication APIs.

---

## **Team Alpha – Backend Database Team**

**Sprint Responsibility:** Database Design and Implementation

**Sprint Tasks:**

1. Create a **Work Breakdown Structure (WBS)** covering all database-related development tasks.
2. Prepare an **Activity Network Diagram (AND)** that shows the logical sequence of database tasks.
3. **Design the application database schema** for the Smart Plot Investment Portal.
4. **Create the database and all required tables.**
5. Write and document the **DDL statements** used for creating tables.
6. Define and document **table relationships**, including:

   * Primary Keys (PK)
   * Foreign Keys (FK)
   * Constraints
7. Prepare **technical documentation** explaining:

   * Database schema
   * Table structures
   * Relationships between tables
8. Conduct a **short technical walkthrough session** with the entire team explaining the database design.

**Expected Deliverables:**

* Database created
* All tables implemented
* Document containing DDL scripts and table relationships
* Presentation/demo for the team

---

## **Team Charlie – Backend Authentication Team**

**Sprint Responsibility:** Backend Signup and Login API Development

**Sprint Tasks:**

1. Coordinate with **Team Alpha** to obtain the finalized **User table structure**.
2. Understand the following from Team Alpha:

   * User table fields
   * Role definitions
   * Required constraints
3. Design and implement **backend API endpoints** for:

   * **User Signup**
   * **User Login**
4. Implement basic **input validation and request handling**.
5. Test APIs using **Postman**.
6. Verify **database connectivity and data insertion** during signup.
7. Validate **user authentication logic** during login.

**Expected Deliverables:**

* Working Signup API
* Working Login API
* Postman test results
* Short explanation/demo of API workflow
